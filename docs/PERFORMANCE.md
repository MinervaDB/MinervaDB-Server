## Storage Engine Optimization

### Granule Size

The `index_granularity` controls how many rows are stored per index mark. Default is 8192.

```sql
CREATE TABLE events (
    event_date Date,
        event_time DateTime,
            user_id UInt64,
                event_type LowCardinality(String),
                    value Float64
                    ) ENGINE = MergeTree()
                    PARTITION BY toYYYYMM(event_date)
                    ORDER BY (event_type, user_id, event_time)
                    SETTINGS
                        index_granularity = 8192,
                            index_granularity_bytes = 10485760;  -- 10MB adaptive granularity
                            ```

                            **Tuning Guidelines:**
                            - High-cardinality point lookups: reduce to 1024–4096
                            - Large analytical scans: keep at 8192 or increase to 16384
                            - Enable adaptive granularity (`index_granularity_bytes`) for mixed workloads

                            ### MergeTree Merge Settings

                            ```xml
                            <!-- config.d/05-mergetree.xml -->
                            <clickhouse>
                              <merge_tree>
                                  <max_bytes_to_merge_at_max_space_in_pool>161061273600</max_bytes_to_merge_at_max_space_in_pool>
                                      <number_of_free_entries_in_pool_to_lower_max_size_of_merge>8</number_of_free_entries_in_pool_to_lower_max_size_of_merge>
                                          <max_parts_in_total>100000</max_parts_in_total>
                                              <parts_to_delay_insert>1000</parts_to_delay_insert>
                                                  <parts_to_throw_insert>3000</parts_to_throw_insert>
                                                      <max_delay_to_insert>1</max_delay_to_insert>
                                                          <enable_mixed_granularity_parts>1</enable_mixed_granularity_parts>
                                                              <concurrent_part_removal_threshold>100</concurrent_part_removal_threshold>
                                                                </merge_tree>
                                                                </clickhouse>
                                                                ```

                                                                ## Primary Key and Sorting Key Design

                                                                The ORDER BY clause defines the sorting key which also determines the primary key by default.

                                                                ### Design Principles

                                                                1. **Place low-cardinality columns first** — filters on these columns prune the most granules
                                                                2. **Time-series data**: `ORDER BY (tenant_id, metric_name, timestamp)`
                                                                3. **Event data**: `ORDER BY (event_date, user_id, event_type, event_time)`
                                                                4. **Log data**: `ORDER BY (service, log_level, timestamp)`

                                                                ### Separating Primary Key from Sort Key

                                                                ```sql
                                                                CREATE TABLE metrics (
                                                                    ts DateTime,
                                                                        host LowCardinality(String),
                                                                            metric LowCardinality(String),
                                                                                value Float64,
                                                                                    tags Map(String, String)
                                                                                    ) ENGINE = MergeTree()
                                                                                    ORDER BY (host, metric, ts)
                                                                                    PRIMARY KEY (host, metric)  -- sparse index on fewer columns for faster lookups
                                                                                    SETTINGS index_granularity = 8192;
                                                                                    ```

                                                                                    ### Analyze Primary Key Usage

                                                                                    ```sql
                                                                                    -- Check if queries use primary key
                                                                                    EXPLAIN indexes = 1
                                                                                    SELECT count() FROM metrics
                                                                                    WHERE host = 'web-01' AND metric = 'cpu_usage' AND ts >= now() - INTERVAL 1 HOUR;
                                                                                    ```

                                                                                    ## Partition Design

                                                                                    ### Best Practices

                                                                                    ```sql
                                                                                    -- Time-based partitioning (most common)
                                                                                    PARTITION BY toYYYYMM(event_date)      -- Monthly: balanced for most use cases
                                                                                    PARTITION BY toYYYYMMDD(event_date)    -- Daily: for high-volume tables
                                                                                    PARTITION BY toYear(event_date)        -- Yearly: for low-volume historical data

                                                                                    -- Multi-dimensional partitioning
                                                                                    PARTITION BY (toYYYYMM(event_date), region)  -- Geographic + time

                                                                                    -- Hash-based for multi-tenant
                                                                                    PARTITION BY cityHash64(tenant_id) % 16
                                                                                    ```

                                                                                    ### Partition Pruning Verification

                                                                                    ```sql
                                                                                    SELECT partition, name, rows, bytes_on_disk, modification_time
                                                                                    FROM system.parts
                                                                                    WHERE database = 'mydb' AND table = 'events' AND active = 1
                                                                                    ORDER BY partition;
                                                                                    ```

                                                                                    ### Partition Management

                                                                                    ```sql
                                                                                    -- Drop old partitions (data retention)
                                                                                    ALTER TABLE events DROP PARTITION '2023-01';

                                                                                    -- Move partition to cold storage
                                                                                    ALTER TABLE events MOVE PARTITION '2023-01' TO VOLUME 'cold';

                                                                                    -- Freeze partition for backup
                                                                                    ALTER TABLE events FREEZE PARTITION '2024-01';
                                                                                    ```

                                                                                    ## Compression and Codecs

                                                                                    ### Codec Selection by Data Type

                                                                                    ```sql
                                                                                    CREATE TABLE timeseries (
                                                                                        ts DateTime CODEC(DoubleDelta, LZ4),           -- Timestamps: DoubleDelta best
                                                                                            value Float32 CODEC(Gorilla, ZSTD(3)),          -- Floating point: Gorilla excellent
                                                                                                counter UInt64 CODEC(Delta(8), ZSTD(1)),        -- Monotonic integers: Delta
                                                                                                    label LowCardinality(String) CODEC(ZSTD(6)),    -- Strings: ZSTD with higher level
                                                                                                        raw_bytes String CODEC(ZSTD(1)),                -- Arbitrary: LZ4 or ZSTD(1)
                                                                                                            sparse_col Nullable(Float32) CODEC(ZSTD(3))
                                                                                                            ) ENGINE = MergeTree()
                                                                                                            ORDER BY ts;
                                                                                                            ```
                                                                                                            
                                                                                                            ### Codec Performance Comparison
                                                                                                            
                                                                                                            | Codec | Compression Ratio | Speed | Use Case |
                                                                                                            |-------|-------------------|-------|----------|
                                                                                                            | LZ4 | Medium | Fastest | Default, general purpose |
                                                                                                            | ZSTD(1) | High | Fast | Better ratio than LZ4 |
                                                                                                            | ZSTD(6-9) | Very High | Medium | Cold/archive data |
                                                                                                            | Delta | Medium | Fast | Monotonically increasing integers |
                                                                                                            | DoubleDelta | High | Fast | Timestamps, time series |
                                                                                                            | Gorilla | Very High | Fast | Floating point time series |
                                                                                                            | FPC | High | Medium | Double precision floats |
                                                                                                            
                                                                                                            ### Column-Level Compression Settings
                                                                                                            
                                                                                                            ```xml
                                                                                                            <compression>
                                                                                                              <case>
                                                                                                                  <min_part_size>10000000000</min_part_size>
                                                                                                                      <min_part_size_ratio>0.01</min_part_size_ratio>
                                                                                                                          <method>zstd</method>
                                                                                                                              <level>3</level>
                                                                                                                                </case>
                                                                                                                                </compression>
                                                                                                                                ```

                                                                                                                                ## Skipping Indexes
                                                                                                                                
                                                                                                                                ### Types and Usage
                                                                                                                                
                                                                                                                                ```sql
                                                                                                                                CREATE TABLE logs (
                                                                                                                                    ts DateTime,
                                                                                                                                        level LowCardinality(String),
                                                                                                                                            message String,
                                                                                                                                                trace_id String,
                                                                                                                                                    user_id UInt64,
                                                                                                                                                        INDEX idx_level level TYPE set(100) GRANULARITY 4,
                                                                                                                                                            INDEX idx_trace trace_id TYPE bloom_filter(0.01) GRANULARITY 1,
                                                                                                                                                                INDEX idx_user user_id TYPE minmax GRANULARITY 4,
                                                                                                                                                                    INDEX idx_msg message TYPE tokenbf_v1(10240, 3, 0) GRANULARITY 1
                                                                                                                                                                    ) ENGINE = MergeTree()
                                                                                                                                                                    PARTITION BY toDate(ts)
                                                                                                                                                                    ORDER BY (level, ts);
                                                                                                                                                                    ```
                                                                                                                                                                    
                                                                                                                                                                    ### Index Type Selection
                                                                                                                                                                    
                                                                                                                                                                    | Index Type | Best For | Cardinality |
                                                                                                                                                                    |------------|----------|-------------|
                                                                                                                                                                    | `minmax` | Numeric ranges, dates | Any |
                                                                                                                                                                    | `set(N)` | Low cardinality equality | < 10,000 |
                                                                                                                                                                    | `bloom_filter` | High cardinality equality, IN | Any |
                                                                                                                                                                    | `tokenbf_v1` | Full-text LIKE/hasToken | Text |
                                                                                                                                                                    | `ngrambf_v1` | Substring LIKE | Text |
                                                                                                                                                                    
                                                                                                                                                                    ### Verify Index Effectiveness
                                                                                                                                                                    
                                                                                                                                                                    ```sql
                                                                                                                                                                    -- Check if index is being used
                                                                                                                                                                    EXPLAIN indexes = 1
                                                                                                                                                                    SELECT * FROM logs WHERE trace_id = 'abc123' AND ts >= today() - 1;
                                                                                                                                                                    
                                                                                                                                                                    -- Index statistics
                                                                                                                                                                    SELECT table, name, type, expr, granularity
                                                                                                                                                                    FROM system.data_skipping_indices
                                                                                                                                                                    WHERE database = 'mydb';
                                                                                                                                                                    ```

                                                                                                                                                                    ## Query Analysis and Profiling
                                                                                                                                                                    
                                                                                                                                                                    ### EXPLAIN Analysis
                                                                                                                                                                    
                                                                                                                                                                    ```sql
                                                                                                                                                                    -- Syntax validation
                                                                                                                                                                    EXPLAIN SELECT user_id, count() FROM events WHERE event_date = today() GROUP BY user_id;
                                                                                                                                                                    
                                                                                                                                                                    -- Logical query plan
                                                                                                                                                                    EXPLAIN PLAN SELECT user_id, count() FROM events WHERE event_date = today() GROUP BY user_id;
                                                                                                                                                                    
                                                                                                                                                                    -- Physical execution pipeline
                                                                                                                                                                    EXPLAIN PIPELINE SELECT user_id, count() FROM events WHERE event_date = today() GROUP BY user_id;
                                                                                                                                                                    
                                                                                                                                                                    -- Full analysis with indexes
                                                                                                                                                                    EXPLAIN indexes = 1, actions = 1
                                                                                                                                                                    SELECT user_id, count() FROM events WHERE event_date = today() GROUP BY user_id;
                                                                                                                                                                    ```
                                                                                                                                                                    
                                                                                                                                                                    ### Query Log Analysis
                                                                                                                                                                    
                                                                                                                                                                    ```sql
                                                                                                                                                                    -- Top slow queries (last 24 hours)
                                                                                                                                                                    SELECT
                                                                                                                                                                        query_id,
                                                                                                                                                                            user,
                                                                                                                                                                                round(query_duration_ms / 1000, 2) AS duration_sec,
                                                                                                                                                                                    formatReadableSize(memory_usage) AS memory,
                                                                                                                                                                                        read_rows,
                                                                                                                                                                                            formatReadableSize(read_bytes) AS read_size,
                                                                                                                                                                                                normalizeQuery(query) AS normalized_query
                                                                                                                                                                                                FROM system.query_log
                                                                                                                                                                                                WHERE
                                                                                                                                                                                                    type = 'QueryFinish'
                                                                                                                                                                                                        AND event_time >= now() - INTERVAL 24 HOUR
                                                                                                                                                                                                            AND query_duration_ms > 5000
                                                                                                                                                                                                            ORDER BY query_duration_ms DESC
                                                                                                                                                                                                            LIMIT 20;
                                                                                                                                                                                                            
                                                                                                                                                                                                            -- Queries with full table scans
                                                                                                                                                                                                            SELECT
                                                                                                                                                                                                                normalizeQuery(query) AS query,
                                                                                                                                                                                                                    count() AS count,
                                                                                                                                                                                                                        avg(query_duration_ms) AS avg_ms,
                                                                                                                                                                                                                            avg(read_rows) AS avg_rows
                                                                                                                                                                                                                            FROM system.query_log
                                                                                                                                                                                                                            WHERE
                                                                                                                                                                                                                                type = 'QueryFinish'
                                                                                                                                                                                                                                    AND event_time >= now() - INTERVAL 1 HOUR
                                                                                                                                                                                                                                        AND ProfileEvents['SelectedFullRanges'] > 0
                                                                                                                                                                                                                                        GROUP BY query
                                                                                                                                                                                                                                        ORDER BY avg_ms DESC
                                                                                                                                                                                                                                        LIMIT 10;
                                                                                                                                                                                                                                        ```
                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                        ### ProfileEvents Deep Dive

                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ## Materialized Views and Projections
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ### Materialized Views for Aggregations
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ```sql
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            -- Source table
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            CREATE TABLE raw_events (
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                ts DateTime,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    user_id UInt64,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        event_type LowCardinality(String),
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            value Float64
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ) ENGINE = MergeTree()
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ORDER BY (event_type, user_id, ts);
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            -- Materialized view for hourly aggregates
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            CREATE MATERIALIZED VIEW mv_hourly_stats
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ENGINE = AggregatingMergeTree()
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ORDER BY (event_type, toStartOfHour(ts))
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            AS SELECT
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                event_type,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    toStartOfHour(ts) AS hour,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        countState() AS cnt,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            sumState(value) AS total_value,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                avgState(value) AS avg_value,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    maxState(value) AS max_value
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    FROM raw_events
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    GROUP BY event_type, hour;
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    -- Query the materialized view
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    SELECT
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        event_type,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            hour,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                countMerge(cnt) AS count,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    sumMerge(total_value) AS total,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        avgMerge(avg_value) AS average,
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            maxMerge(max_value) AS maximum
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            FROM mv_hourly_stats
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            WHERE hour >= toStartOfHour(now() - INTERVAL 24 HOUR)
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            GROUP BY event_type, hour
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ORDER BY hour;
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ```
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            ### Projections for Alternative Sort Orders
